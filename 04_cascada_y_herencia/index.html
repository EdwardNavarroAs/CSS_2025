Cascada, herencia y especificidad

Comprende cómo el navegador decide qué estilos aplicar en caso de conflicto.

1. ¿Qué es la cascada?

La cascada es el sistema que usa CSS para resolver qué regla aplicar cuando varias afectan al mismo elemento. Se basa en el origen del estilo, el orden en que aparecen y si hay conflictos entre reglas.

2. ¿Qué es la herencia?

La herencia permite que algunos estilos como el color o la tipografía se transmitan desde un elemento padre a sus hijos. Sin embargo, no todas las propiedades se heredan por defecto (por ejemplo, los márgenes o los bordes).

3. ¿Qué es la especificidad?

La especificidad es una medida que determina qué regla tiene más peso cuando varias se aplican al mismo elemento. Se calcula así:

  • Selector de etiqueta → peso = 1
  • Selector de clase o pseudoclase → peso = 10
  • Selector de ID → peso = 100
  • Estilo en línea → peso = 1000
  • !important → sobrescribe todo

¿Qué color ves aquí? El ID tiene más peso que la clase o la etiqueta.

Este párrafo usa estilo en línea, que tiene el mayor peso.

4. Conflictos entre reglas

Cuando hay múltiples reglas con igual especificidad, se aplica la que esté más abajo. Pero si una regla usa !important, ganará sobre cualquier otra, incluso sobre estilos en línea.
